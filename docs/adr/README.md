# Architecture Decision Records (ADRs)

ADRs capture durable architecture choices and why they were made.

## Conventions

- File format: `NNNN-short-title.md`
- Status values: `proposed`, `accepted`, `superseded`
- Keep one decision per ADR
- Link related ADRs and the to-do item that introduced the decision

## Template

Use this skeleton:

```md
# ADR NNNN: Title

- Status: proposed|accepted|superseded
- Date: YYYY-MM-DD
- Related: links to docs/to_do.md, PRs, issues

## Context

## Decision

## Consequences

## Alternatives considered
```
