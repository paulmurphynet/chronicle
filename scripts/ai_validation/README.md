# Scenario validation scripts

Run a **rule-based scenario driver** (no AI/LLM) through a Chronicle workflow and score the result. Used for Phase 1 of the [Scenario validation and Learn roadmap](../../docs/ai-validation-and-learn-roadmap.md).. The driver is a deterministic script (ChronicleSession + fixed steps); scenarios and sample generation use no Ollama, no API keys, and no live model. All runs are fully scriptable and repeatable; CI and local dev can run them without any model. The Learn sample (`sample.chronicle`) is produced by `scripts/generate_sample_chronicle.py` (same deterministic pipeline).

**Gold for Learn (Phase 2):** The Journalism conflict scenario (London vs Paris) is the same content as the app's "Try sample" / "Load Journalism sample" exemplar. The canonical gold file is `frontend/public/sample.chronicle`, generated by `PYTHONPATH=. python3 scripts/generate_sample_chronicle.py`. The Learn page loads this so users can view a completed investigation before following the steps. Regenerate the sample when you change the scenario; `run_agent.py` produces equivalent content for the scorer.

**Run (from repo root):**

```bash
# Run one scenario (session-based; no API server)
PYTHONPATH=. python3 scripts/ai_validation/run_agent.py --scenario journalism_conflict
# or: --scenario legal_conflict

# Score the last export (or pass path)
PYTHONPATH=. python3 scripts/ai_validation/score_run.py [path/to/export.chronicle]

# Run all scenarios and report pass/fail (Phase 6: regression)
PYTHONPATH=. python3 scripts/ai_validation/run_all_verticals.py
```

**Scenarios:** Nine scenarios across all seven Learn verticals. **Journalism** (2): `journalism_conflict` (London vs Paris, tension), `journalism_single_claim`. **Legal** (2): `legal_conflict` (delivery dates, tension), `legal_single_claim`. **Compliance, Fact-checking, Internal investigations, Due diligence, Academic** (1 each): `compliance_single_claim`, `fact_checking_single_claim`, `internal_investigations_single_claim`, `due_diligence_single_claim`, `academic_single_claim` — each is one doc, one claim, one support, no tension (`require_tension: false`). Use `--scenario <id>`; run all with `run_all_verticals.py`.

**Scorer:** Runs `chronicle-verify` on the export, then rubric checks (investigations >= 1, claims >= 2, support links >= 1, tensions >= 1). Exit 0 only if all pass. **On failure** the scorer writes `reports/last_run.json` and `reports/last_run.md` (scenario, conformance/rubric results, summary, and a suggestion line you can add to to-do). Use `--no-report` to skip writing. After a failed run, check the report and fix the app, playbook, or scenario as needed.

**Phase 9 (optional) — Learn content from runs:** Run with `--trace` to write a run trace to `reports/traces/<scenario_id>.json`. Then run `PYTHONPATH=. python3 scripts/ai_validation/propose_learn_from_trace.py` to get markdown suggestions for adding or aligning `example` / tips in `chronicle/api/static/learn/guides.json`. Review and approve before editing the guides.
